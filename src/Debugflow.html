附加与启动
Attach 进程 -> 写入硬件调试寄存器 (下发断点) -> 恢复全速运行
核心监控循环 (CE 死循环逻辑)
[A] 事件触发
(硬件断点命中，内核抛出 SIGTRAP，暂停线程)
[B] 获取线程上下文
(读取寄存器，解析发生写操作的地址及当前 PC)
[C] 设置线程上下文
(修改 CPU 标志位，如 ARM64 PSTATE.SS 或 x86 TF 位)
[D] 取消断点
(临时清空硬件调试寄存器：撤防)
[E] 发送继续调试
(内核唤醒线程，因设置了 TF/SS，CPU 准备单步)
[F] CPU 执行原指令
(仅执行一条指令后，再次触发单步 Trap 事件)
[G] 断点回挂
(将监控地址重新写入硬件调试寄存器：重新布防)
[H] 发送继续调试
(清理单步标志，内核唤醒线程，全速运行)
用户主动中止
清理与分离
永久移除硬件断点 -> Detach 分离调试器 -> 目标完全自由
